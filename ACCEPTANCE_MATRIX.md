# QuantumZ Phase 15 Acceptance Matrix

Date: 2026-06-21

Purpose: final implementation hand-off checklist for Phases 0-15. This matrix records what is already verified by repository evidence and the build gate, and what must still be validated on a real arm64 Android device before release acceptance.

## Verification Legend

- **Pass — Static/Build**: verified from source code, project configuration, repository artifacts, or the final build output recorded below.
- **Requires Real Device Validation**: implementation evidence exists, but the requirement is behavioral, platform, audio, UX, or end-to-end and must be exercised on a physical arm64 Android device.
- **Partial / Follow-up**: static evidence is incomplete or identifies a remaining verification gap.

## Final Hand-Off Checklist for Real Device QA

1. Fresh install with no settings data opens `SetupPage`; remote setup saves endpoint/model/key and navigates to main.
2. On-device setup shows checklist, installs/selects required whisper + llama artifacts, and blocks completion until native runtimes/models are ready.
3. Navigation: Main → Settings → Back; Main → Memory → Back; Details panel open/close; rapid panel toggles with no `SemaphoreFullException`.
4. Remote internet-on loop: Listen → speak → transcription → LLM response → spoken response; wrong/secured endpoint shows friendly status, no raw popup.
5. On-device airplane-mode loop: Listen → speak → local Whisper transcription → local Llama reply → Android TTS speech; no Piper/empty-byte crash.
6. Settings persistence: edited endpoint/model/tool iteration/audio values survive app restart.
7. Stability: repeated listening start/stop leaves Android mic indicator off and logs no unobserved task crashes.
8. UI/UX: Main, Settings, Setup, Memory, and Debug pages scroll and remain readable at large font size; all interactive controls expose accessibility labels.
